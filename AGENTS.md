# AGENTS.md

Notes for OpenCode sessions working in this repo. High-signal facts only.

## What this repo is

Research repo on detecting whether a model has unlearned a domain, using only
the model itself (no original, no shadow models) against the TOFU suite
(`locuslab/TOFU`) and OpenUnlearning's Llama-3.2-1B-Instruct checkpoints.

- No CI, no tests, no lint, no typecheck, no codegen. Do not invent commands.
- Only manifest is `pip install -r requirements.txt`. Python 3.12, torch 2.8 + CUDA.
- `PROPOSAL.md` = thesis/design. `experiments/EXPLANATION.md` = canonical runbook — read it before running anything.

## Layout (which dirs matter)

- `experiments/` — **current main pipeline.** `common.py` (model registry + stats + I/O), `spectral.py` (E1–E3, GPU forward passes), `weights.py` (E4–E5, weights only), `detect.py` (E6, combines prior outputs), `mia.py` (E7, MINT-inspired token-level MIA metrics), `samples.py` (E8, text completions for qualitative inspection). Run from this dir.
- `fast_experiments/` — **legacy**, older Phi-1.5 / Zephyr-7b probe approach (`reproduction.py` for CUDA, `*_xpu.py` for Intel dGPUs, `reproduction_tofu.ipynb`). Not the current line of work; don't update unless explicitly asked.
- `logs_tofu/` — pre-cached `.npz`/`.json` TOFU splits and a prior MLP-probe run. Read-only cache; not regenerated by `experiments/`.
- `results.json` — single Zephyr-7b RMU/MMLU run from the legacy pipeline.
- `experiments/results/` — output of the current pipeline (gitignored, untracked). Subdirectories: `spectral/`, `weights/`, `detection/` (original pipeline); `spectral_v2/`, `weights_v2/`, `detection_v2/` (v2 with chat-template); `spectral_v2_seed99/` (seed-99 ablation); `mia/`, `samples/` (E7/E8); `*_phi/` variants (Phi-1.5 models, separate model set).

## Canonical commands (from experiments/EXPLANATION.md)

Run from the repo root with `workdir=/root/unlearning_fingertips/experiments`:

```
# Day 1: spectral (E1/E2/E3), ~5 min/model on 1B
# IMPORTANT: requires HF_HOME=/root/.cache/huggingface/ HF_HUB_DISABLE_XET=1
# (see GOTCHA section — using the default HF cache triggers phantom "model not
# found" errors once quota fills after a few downloads).
for m in original retain rmu npo graddiff altpo undial idknll; do
  python spectral.py --model $m --output_dir results/spectral/
done

# Day 2: weights (E4/E5), ~8 min/model (full SVD on MLP matrices)
for m in original retain rmu npo graddiff altpo undial idknll; do
  python weights.py --model $m --output_dir results/weights/
done

# Day 3: detection (E6) — must run AFTER spectral+weights complete for all 8
python detect.py --spectral_dir results/spectral/ --weight_dir results/weights/ --output_dir results/detection/

# Day 4: MINT-style MIA metrics (E7), ~5 min/model
for m in original retain rmu npo graddiff altpo undial idknll; do
  python mia.py --model $m --output_dir results/mia/
done

# Day 5: model output samples (E8), ~2 min/model (50 samples)
for m in original retain rmu npo graddiff altpo undial idknll; do
  python samples.py --model $m --output_dir results/samples/ --num_samples 50
done
```

Requirements:
- `spectral.py` needs a CUDA GPU (200 forget + 200 retain prompts, chat-template formatted).
- `weights.py` is weights-only; full SVD takes ~8 min per model in CPU-bound numpy.
- `detect.py` globs `*_spectral.json` / `*_weights.json` in its input dirs. If you rename a `--model_tag`, both pipelines must use the same tag or detect.py silently drops that model.
- `spectral.py` now accepts `--seed N` (default 42) and `--raw_prompts` (legacy `Question:..\\nAnswer:` format, ablation only — do NOT use for primary runs; the models were trained with the chat template, raw prompts mask the unlearning signal).
- `mia.py` and `samples.py` need a CUDA GPU (forward passes / generation).

## What got changed in the v2 pipeline refactor

- `common.py`: `load_tofu_prompts` now returns raw questions; `format_prompts(questions, tokenizer, raw=False)` applies the tokenizer's chat template (Llama-3 default). `collect_hidden_states` uses left-padding (set in `load_model`) so the last real token is always at position `-1` regardless of batch composition. Bumped `max_length` to 512 (chat-template prompt frames are larger than raw "Question: \\nAnswer:"). Added `spectral_metrics(W)` returning `(stable_rank, sigma_max, sigma_min)`.
- `spectral.py`: consumes `format_prompts`; added `--seed N` and `--raw_prompts` flags; saves `seed` and `raw_prompts` in the JSON.
- `weights.py`: records per-matrix `sigma_max` and `sigma_min` alongside stable_rank; `detect_anomalies` runs on `mean_sigma_max` by default (`--stat_key` CLI override). Plot now shows σ_max by layer.
- `detect.py`: NEW interior-only layer range (skip embedding + last 2 hidden states, which on Llama-3 carry a universal pre-output collapse that masks the signal). NEW per-model scores: `localized_dip`, `localized_kink`, `shape_anomaly`, `cosine_anomaly`. Per-statistic ROC AUC is printed at the end of detect. `combined` is dominated by `cosine_anomaly` (0.65 weight).
- The original pipeline output (raw prompts, global-min Δerank, stable_rank weights only) lives unchanged in `experiments/results/` for diff reference; the refactor writes to `spectral_v2/`, `weights_v2/`, `detection_v2/` (gitignored) so old runs are not silently overwritten.

## GOTCHA: HF cache disk-quota error masquerades as "model not found"

`HF_HOME=/workspace/.cache/huggingface/` lives on a shared network mount
(`mfs#...:/workspace`, ~1PB, per-user-quota'd). The `hf-xet` downloader
(`HF_XET_HIGH_PERFORMANCE=1`, `hf-xet 1.5.1`) errors with:

```
RuntimeError: ... IO Error: Disk quota exceeded (os error 122)
OSError: Can't load the model for 'open-unlearning/...'. ... make sure '...' is the correct path
```

The wrapping `OSError` quotes huggingface_hub's stale "wrong repo id" message —
don't chase the HF path. Symptom signature: the first few models download &
cache fine, then the rest fail identically. Real cause: per-user quota on the
network share is exhausted by the accumulated xet cache.

Fix options (verify space first with `df -h /workspace`):
- Move cache off the share: `export HF_HOME=/root/.cache/huggingface/` (local overlay has ~28G free, enough for the 8 ~2GB 1B checkpoints one at a time).
- Disable xet: `export HF_HUB_DISABLE_XET=1` and `unset HF_XET_HIGH_PERFORMANCE` — uses plain HF transfer, sometimes avoids the per-file reconstruction quota trigger.
- Clear stale xet chunks: `rm -rf /workspace/.cache/huggingface/xet` (safe; re-downloads what's missing).

After the cache is healthy the download resumes without touching HF repo ids.

## GOTCHA: `idknll` checkpoint discrepancy between `common.py` and `EXPLANATION.md`

| Source | `idknll` HF id ends in |
|---|---|
| `experiments/common.py` (committed) | `IdkNLL_lr3e-05_alpha5_epoch10` |
| `experiments/EXPLANATION.md` table | `IdkNLL_lr4e-05_alpha5_epoch10` |

**Both ids return HTTP 200 on `/api/models/...`** — both checkpoints exist on HF.
Don't "fix" one to match the other without confirming which checkpoint the
researcher intends; the two `lr` values are different unlearning runs. Pick one,
run spectral/weights with it, and note the choice in results. The actual code
uses `common.py` — `EXPLANATION.md` is stale.

  (or with `experiments/` on `PYTHONPATH`). Imports use `from common import ...`.

## Doing verification here

There is no test suite. To verify edits to the pipeline:
1. Re-run `spectral.py` / `weights.py` on one model (e.g. `--model rmu`) and
   diff the resulting JSON against the prior run in `experiments/results/` —
   statistics are deterministic for a fixed seed (built-in `seed=42` in
   `load_tofu_prompts`), modulo float nondeterminism on GPU.
2. Re-run `detect.py` and check `results/detection/*.png` regenerate and
   `scores.csv` matches prior shape (8 rows).
`AGENTS.md` never substitutes for running these — there's nothing else.
