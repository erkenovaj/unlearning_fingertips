#Can an adversary identify unlearned domains from a single open-weight model?
Research question: Given full weight access to a model but no access to the original pre-unlearning version, can an adversary detect which domains (if any) have been unlearned, by spotting traces that distinguish unlearned domains from domains the model genuinely never knew?
Rough setup:  Use TOFU's unlearning checkpoints. Take the full model (knows everything), unlearned models (multiple methods via OpenUnlearning checkpoints), and the retain-only model (genuinely never saw forget-set data). Search for any detectable differences between unlearned domains and genuinely unknown domains using the full range of signals that open-weight access enables: output-level statistics (calibration, entropy, self-consistency), activation-level analysis (spectral fingerprints via SVD following Chen et al.'s methodology), or weight-level inspection (comparing parameter distributions across layers). First establish whether any signal distinguishes unlearned from genuinely unknown domains. Then test whether that signal is usable in practice (can an adversary scanning across domains identify the unlearned ones without knowing the ground truth?).
What good looks like: If any signal reliably distinguishes unlearned from genuinely unknown domains, that's a practical vulnerability because an adversary can cheaply identify where to focus costly finetuning attacks. If unlearned domains are indistinguishable from genuine ignorance across all signals, that's reassuring for deployment. Either result directly informs how much the field should trust current unlearning methods.
Key resources:
450+ unlearning model checkpoints across many models on the TOFU dataset.
